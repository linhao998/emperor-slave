# Emperor–Slave Card Game（皇帝奴隶卡牌博弈游戏）

## 项目简介

Emperor–Slave 是一款基于经典博弈模型的 **网页端单机卡牌小游戏**，以「最小可行产品（MVP）」为核心原则进行设计与实现。

本项目重点不在于复杂技术，而在于展示：

- 完整、可落地的产品需求分析（PRD）
- 清晰的功能拆解与优先级判断
- 与 AI 协作完成产品落地的能力
- 基础工程与项目管理能力

项目最终目标是：  

 **在有限时间内，完成一个“可玩、可上线、可展示”的产品闭环**。

---

## 项目亮点

-  明确的 MVP 边界与版本规划  
-  以流程图 + 原型约束 AI 行为，避免“AI 乱改设计”  
-  AI 作为“工程协作者”而非“黑盒写代码工具”  
-  预留完整的数据埋点与分析思路  
-  支持一行命令构建与上线（Web）

---

## 目录结构说明

本项目采用 **“展示优先 + 工程分层”** 的目录设计，兼顾可读性与开发可维护性。

```text
emperor-slave-game/
├── assets/        #  供 AI / 前端直接取用的素材资源
├── docs/          #  补充文档（流程图、表格、说明等）
├── prd_html/      #  PRD 原型（HTML，可直接浏览）
├── prd_rp/        #  PRD 原型源文件（Axure RP）
├── src/           #  游戏源码
├── builds/        #  构建产物与上线信息
└── README.md
```

## 如何快速了解本项目

1. 阅读 PRD（HTML 或RP文档）

2. 访问Demo（在线版见主目录，apk见 builds 目录）

3. 查看目录结构，理解项目组织方式

4. 如需深入，可查看 docs / src
